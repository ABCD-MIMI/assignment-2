Scope of var, let, and const:
var: It is function scoped, meaning it's only available within the function it's declared in. If it's not in any function, then it is globally scoped.
let: Unlike var, let is block-scoped. Here, its scope is limited within the block, statement, or expression where it is defined.
const: Just like let, const is also block-scoped. However, itâ€™s a constant reference to a value, meaning you can't reassign it to a new value.
Use Cases of null and undefined:
null: null represents the intentional absence of any object value. It's often used when you want to express the concept of "no value" or "no object" intentionally.
undefined: undefined is given to a variable that has been declared but not assigned a value, whether it's object, function, number or any other type.
REST API:
REST (Representational State Transfer) API is a set of principles for designing network applications. It utilizes HTTP protocols for creating, reading, updating or deleting data (known as CRUD operations). It makes interactions between clients and services easy since it uses a streamlined, stateless, cacheable communication protocol that is standardized all over the world.
Examples of these standard HTTP methods within RESTful principles include:
1.GET for reading data
2.POST for creating data
3.PUT for updating data
4.DELETE for deleting data
